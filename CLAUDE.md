# Grid to Green

AI-powered F1 race briefings that synthesize prediction market odds, qualifying telemetry, circuit history, and weather into opinionated pre-race previews. Generated by Claude, deployed on Vercel.

## Tech Stack

- Next.js 16, React 19, TypeScript
- Tailwind CSS v4 + shadcn/ui
- Vercel KV (Upstash Redis) ﻗ falls back to local `.data/kv.json` in dev
- Anthropic Claude API for briefing generation

## Conventions

### Text Generation

All text generation via Claude must follow these guidelines:

- **Never use em dashes (ﻗ)** in any generated text. Replace em dashes with commas or separate sentences for clarity.
- **Hyphens are only for compound words** like "pre-race", "head-to-head", or "post-qualifying". Do not use hyphens to replace em dashes mid-sentence.
- Tone: confident, opinionated, slightly irreverent, never boring. Be an analyst who uses data to make arguments, not a stats reciter.
- Every number cited should serve a point, not just fill space.

These guidelines are enforced in the system prompts in `lib/claude.ts` (`SYSTEM_PROMPT` and `PREVIEW_SYSTEM_PROMPT`).

### Caching

All data-fetching functions use the Next.js 16 `"use cache"` directive with explicit lifetime and tags:

```ts
async function getData() {
  "use cache";
  cacheLife("max");      // "max" | "hours" | { revalidate: seconds }
  cacheTag("briefing");  // tag for on-demand revalidation
  return fetchData();
}
```

**Cache tags in use:** `"briefing"`, `"odds-history"`, `"polymarket-slug"`, `"upcoming"`

Caching is applied in page-level wrapper functions, NOT in `lib/kv.ts` or other data modules.

### Error handling

External data failures degrade gracefully ﻗ `.catch(() => null)` or `.catch(() => [])`. Missing data results in partial briefings, never thrown errors.

### Components

- Server Components by default; `"use client"` only for interactivity (charts, share buttons)
- Dark theme only (hardcoded `.dark` class on `<html>`)
- Custom CSS vars: `--color-racing-red`, `--color-terminal-green`, `--color-terminal-amber`
- Font classes: `font-heading` (Bebas Neue), `font-sans` (Geist), `font-mono` (Geist Mono)
- Container pattern: `mx-auto max-w-7xl px-4`

### Types

All shared types live in `types/index.ts`. Key interfaces:
- `Briefing` / `BriefingMeta` ﻗ stored briefing and lightweight index entry
- `BriefingContext` ﻗ assembled data passed to Claude for generation
- `GeneratedBriefing` ﻗ Claude's structured JSON output

### Utilities

- `getBaseUrl()` in `lib/utils.ts` ﻗ use this instead of inline `process.env.NEXT_PUBLIC_BASE_URL` reads
- `fetchWithRetry()` in `lib/fetch-utils.ts` ﻗ retry with exponential backoff, 429 handling
- `CIRCUIT_MAP` in `lib/circuits.ts` ﻗ OpenF1 circuit name to Jolpica ID mapping

## External APIs

| API | Auth | Purpose |
|-----|------|---------|
| OpenF1 (`api.openf1.org`) | None | Meetings, qualifying, weather, drivers |
| Jolpica (`api.jolpi.ca/ergast/f1`) | None | Standings, circuit history, race results |
| Polymarket (`gamma-api.polymarket.com`) | None | Race winner odds, price history |
| Open-Meteo (`api.open-meteo.com`) | None | Race day weather forecasts |
| Anthropic Claude | `ANTHROPIC_API_KEY` | Briefing text generation |

## API Routes

All routes require `Authorization: Bearer ${CRON_SECRET}`.

- `POST /api/generate-briefing` ﻗ generate full briefing (post-qualifying)
- `GET /api/cron` ﻗ generate briefing for next upcoming race
- `POST /api/generate-previews` ﻗ generate preview briefings for future races

## Required Environment Variables

- `ANTHROPIC_API_KEY` ﻗ Claude API

- `KV_REST_API_URL` + `KV_REST_API_TOKEN` ﻗ Upstash Redis (omit for local dev)
